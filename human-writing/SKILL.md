# Human Writing - 活人感写作心法

> AI 时代，你的不完美才是最坚固的护城河

## 快速开始

```bash
# 写作前自检
human-writing --check --topic="你的选题"

# 使用故事模板
human-writing --template=story

# 完整的 HKR 评分
human-writing --score article.md
```

## 核心设计：三层心法

| 层级 | 作用 | 解决的问题 |
|------|------|-----------|
| **心法** | 成为"人" | 为什么我的文章像机器写的？ |
| **战略** | 选题方向 | 写什么才能有人看？ |
| **技法** | 表达方式 | 怎么写才能打动人？ |

## 心法层：成为"人"

### 1. 活人感 - AI 无法复现的护城河

**AI 能做的**：
- 整合知识
- 模仿文风
- 生成结构

**AI 做不到的**：
- 你的体温、心跳
- 你的回忆、伤疤
- 你的"我以为"

**实践方法**：
```
❌ 理性的第三方视角：
《AI时代的信息差商业模式探究》

✅ 你的亲身经历：
《在淘宝花9.9元买DeepSeek的魔幻体验》
- 一开始的好奇
- 付款后的震惊
- 看评论区时的百感交集
- 对这个世界的感同身受
```

### 2. 真诚 - 永远的必杀技

**原则**：可以不说，但绝不欺骗

**自检清单**：
- [ ] 这个观点我真心认同吗？
- [ ] 这个产品我真的觉得好用吗？
- [ ] 如果评论区有人质疑，我能站得住脚吗？

> "建立一个好名声需要20年，毁掉它只需要5分钟。" - 巴菲特

### 3. 价值观 - 内容底线

**不是每波流量都要追**

当热点与你的价值观冲突时：
```
短期：一篇爆款，但读者觉得"你变了"
长期：人设崩塌，IP价值归零

vs

短期：错过流量
长期：守住品牌价值，读者有归属感
```

**你的价值观是什么？**（写下来）
```
我的内容底线：
1. ________________
2. ________________
3. ________________
```

## 战略层：选题决定80%的生死

### 选题公式

```
好选题 = 你的专业领域 + 读者的普遍兴趣 + 当下的时间节点
```

| 维度 | 问题 | 示例 |
|------|------|------|
| **专业** | 这个问题我的研究超过90%读者吗？ | AI是我的专业领域 |
| **兴趣** | 这是所有人都关心的事吗？ | 生命健康 |
| **时机** | 现在写有什么契机吗？ | 达摩院刚发新论文 |

**三者叠加 = 势能**

### 专家心态

**先成为专家，再开始写**

不需要博士学位，只需要：
- 在这个**具体问题**上，研究超过90%读者
- 承认自己的无知，不硬写不懂的领域
- 把相关资料、论文、产品都用一遍

**研究清单**：
```
□ 核心论文/报告 3-5 篇
□ 竞品分析（别人怎么写的）
□ 产品/服务实际体验
□ 评论区真实用户反馈
□ 常见问题与误解
```

**敬畏之心**：
> "评论区里一定有真正的专家。不要给他们挑错的机会。"

## 技法层：说到别人心里去

### 故事思维 - 不是论文，是冒险

**论文思维 vs 故事思维**

| 论文 | 故事 |
|------|------|
| 结构完整 | 引人入胜 |
| 逻辑严谨 | 情感共鸣 |
| 客观中立 | 主观体验 |
| 人脑难记 | 人脑好记 |

**万能故事结构**：

```
冲突：我遇到了什么麻烦？
  ↓
探索：我怎么寻找解决方案？
  ↓
发现：怎么遇到这个产品/观点？
  ↓
高潮：最让我惊艳的瞬间？
  ↓
结局：解决了什么麻烦？生活有什么改变？
```

**示例**：
```
冲突：每天要花2小时整理投研信息
探索：试过Excel、Notion、各种工具
发现：朋友推荐了OpenClaw
高潮：发现它能自动抓取龙虎榜、定时推送
结局：现在每天10分钟看简报，决策效率翻倍
```

### 读者画像 - 聪明、有钱、很忙

**你的读者是**：

**1. 聪明的人**
- 不要讲废话
- 不要把读者当傻子
- 相信他们能听懂
- 把复杂的事讲清楚，而不是简单的事反复说

**2. 有钱（选择很多）的人**
- 他关注了100个博主
- 读你的文章 = 放弃刷剧/短视频
- 你的内容必须**值得**这个机会成本

**3. 很忙的人**
- 结构清晰，一眼抓重点
- 多用加粗、短段落
- 允许"一目十行"也能有收获

### HKR 评分标准

写完用这三条检查：

| 维度 | 含义 | 自检问题 |
|------|------|---------|
| **H** (Hook) | 愉悦感/好奇心 | 开头能勾住人吗？有"卧槽还有这种事？"的感觉吗？ |
| **K** (Knowledge) | 知识/信息 | 读者能学到新东西吗？有"原来是这样"的收获吗？ |
| **R** (Resonance) | 共鸣 | 能戳中读者吗？有"对对对我也是这么想的"的感觉吗？ |

**评分表**：
```
H (Hook):     ⭐⭐⭐⭐⭐ /5
K (Knowledge): ⭐⭐⭐⭐⭐ /5  
R (Resonance): ⭐⭐⭐⭐⭐ /5

总分: __/15

改进建议：
H不足 → 换个更有冲击力的开头
K不足 → 补充硬核信息或独特洞察  
R不足 → 加入更多个人体验或情感描写
```

## 与 ai-flavor-remover 的协同

**写作流程**：

```
1. 选题阶段
   └─> human-writing --check --topic="xxx"
       └─> 确认：专业+兴趣+时机 ✅

2. 写作阶段  
   └─> human-writing --template=story
       └─> 按冲突→探索→发现→高潮→结局 写作

3. 润色阶段
   └─> ai-flavor-remover --mode=medium
       └─> 去除机械句式，注入口语化表达

4. 质检阶段
   └─> human-writing --score article.md
       └─> HKR 评分，确保 H/K/R 都达标
```

**两者的互补关系**：

| 技能 | 解决什么问题 | 作用时机 |
|------|-------------|---------|
| **human-writing** | 从0到1写出好内容 | 写作前/中/后 |
| **ai-flavor-remover** | 让内容读起来更自然 | 写作后 |

## CLI 使用指南

### 选题检查

```bash
human-writing --check --topic="Claude Constitution"

输出：
[选题分析]
专业度: ★★★★☆ (AI Agent架构是你的专业)
兴趣度: ★★★★☆ (AI安全是普遍关注话题)
时机性: ★★★★★ (Claude刚发布Constitution)

建议: 这是一个好选题，三者叠加有势能。
      注意结合你的OpenClaw使用经验，
      提供独特视角。
```

### 故事模板

```bash
human-writing --template=story --input="产品体验"

输出：
冲突：_______________
探索：_______________
发现：_______________
高潮：_______________
结局：_______________
```

### HKR 评分

```bash
human-writing --score article.md

输出：
[HKR 评分]
H (Hook):     ★★★★☆ 4/5 - 开头有冲击力，但可以更强
K (Knowledge): ★★★★★ 5/5 - 信息密度高，有独特洞察
R (Resonance): ★★★☆☆ 3/5 - 建议加入更多个人体验

总分: 12/15

改进建议：
1. R不足：在"Constitution设计"部分，
         加入你作为OpenClaw用户的第一反应
2. H可提升：开头用具体场景代入，
          而不是直接讲概念
```
